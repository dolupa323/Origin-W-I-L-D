# Developer Workflow Reference

이 문서는 Origin-WILD 프로젝트에서 채팅 세션이 바뀌어도 유지되는 작업 원칙 참고 문서입니다.

## 1. 전략 수립 원칙

모든 요구사항은 아래 순서로 처리합니다.

1. 동시에 3가지 전략을 설계한다.
2. 각 전략을 장점, 단점, 예상 비용으로 평가한다.
3. 가장 비합리적인 전략 1개를 가지치기한다.
4. 남은 대안 중 최적 1개만 선택해 구현한다.

## 2. 완료 후 안전 점검 원칙

구현이 끝나면 즉시 아래를 수행합니다.

1. 가장 치명적인 논리적 약점 또는 런타임 에러 가능 지점을 식별한다.
2. 해당 문제를 스스로 수정한다.
3. 수정 반영된 최종 버전을 전달한다.

## 3. 모호한 요청 처리 원칙

요청이 불명확하면 바로 작업하지 않고, 아래 3가지를 포함해 재요청한다.

- 범위 (무엇을 어디까지 바꿀지)
- 제약 (성능, 호환성, 스타일, 마감 등)
- 기대 산출물 형식 (코드, 문서, 테스트, 설명 형태)

## 4. \_agent 스킬 사용 인지

이 프로젝트에는 [\_agent/skills](../_agent/skills/) 경로에 스킬들이 존재하며, 작업 계획/분석/구현 시 참조 가능한 리소스로 간주합니다.

- 스킬은 보조 수단이며, 현재 저장소 코드/문서 컨텍스트를 우선한다.
- 스킬 사용 시에도 최종 결정은 프로젝트 요구사항과 안정성 검증 결과를 기준으로 한다.
