| 기능        | 키           | 설명                                     | 상태    |
| ----------- | ------------ | ---------------------------------------- | ------- |
| 스프린트    | Shift (홀드) | 이동속도 1.6배, 스태미나 소모            | ✅ 완료 |
| 구르기/회피 | Ctrl         | 12스터드 이동, 0.25초 무적, 쿨다운 0.8초 | ✅ 완료 |
| 스태미나    | -            | 자동 회복, HUD 표시                      | ✅ 완료 |

**관련 파일**:

- `StaminaService.lua` (서버) - 스태미나 관리, 무적 프레임
- `MovementController.lua` (클라이언트) - 입력 처리, 서버 요청
- `Balance.lua` - 스태미나/이동 상수 정의

**밸런스 설정** (`Balance.lua`):

```lua
STAMINA_MAX = 100              -- 최대 스태미나
STAMINA_REGEN = 8              -- 초당 회복량
STAMINA_REGEN_DELAY = 1.5      -- 사용 후 회복 시작 딜레이

SPRINT_SPEED_MULT = 1.6        -- 스프린트 속도 배율
SPRINT_STAMINA_COST = 12       -- 초당 소모
SPRINT_MIN_STAMINA = 10        -- 시작 최소 스태미나

DODGE_STAMINA_COST = 25        -- 구르기 1회 소모
DODGE_COOLDOWN = 0.8           -- 쿨다운
DODGE_DISTANCE = 12            -- 이동 거리
DODGE_DURATION = 0.4           -- 소요 시간
DODGE_IFRAMES = 0.25           -- 무적 시간
```

## 4. 상세 기획

### 4.1 제작(Crafting) UI

**목적**: 아이템 제작 레시피 표시 및 제작 요청

**위치**: 화면 중앙 팝업 (C키 또는 작업대 상호작용)

**레이아웃**:

```
┌─────────────────────────────────────────────┐
│ 🔨 제작                              [X]   │
├────────────────┬────────────────────────────┤
│  [카테고리]    │  [레시피 상세]             │
│  ■ 도구       │  ┌─────┐                   │
│  ■ 무기       │  │ 아이콘 │ 돌 곡괭이      │
│  ■ 건축       │  └─────┘                   │
│  ■ 기타       │  필요 재료:                │
│               │  - 돌 3개 ✓               │
│  [레시피 목록] │  - 나무 2개 ✓             │
│  ├ 돌 곡괭이   │  - 섬유 5개 ✓             │
│  ├ 돌 도끼     │                           │
│  ├ 나무 클럽   │  제작 시간: 3초           │
│  └ ...        │  [  제작하기  ]            │
└────────────────┴────────────────────────────┘
```

**기능 요구사항**:

- [ ] 카테고리별 레시피 필터링
- [ ] 제작 가능 여부 시각적 표시 (재료 부족 시 회색)
- [ ] 재료 보유량 실시간 표시
- [ ] 테크 레벨 잠금 표시
- [ ] 제작 진행 프로그레스 바
- [ ] 제작 대기열 표시 (작업대 사용 시)

**연동 API**:

```lua
-- CraftController
CraftController.getAvailableRecipes()
CraftController.getRecipeDetails(recipeId)
CraftController.canCraft(recipeId)
CraftController.requestCraft(recipeId, amount)

-- RecipeService (서버)
RecipeService.GetUnlockedRecipes(player)
RecipeService.GetRecipeData(recipeId)
```

## 6. 기술적 고려사항

### 🔧 구현 패턴

1. **모듈화**: 각 UI를 별도 서브모듈로 분리

   ```
   UIManager/
   ├── HUD.lua
   ├── Inventory.lua
   ├── Crafting.lua
   ├── TechTree.lua
   ├── Shop.lua
   ├── Palbox.lua
   └── Build.lua
   ```

2. **이벤트 기반**: 컨트롤러 이벤트에 UI 반응

   ```lua
   InventoryController.onChanged(function()
       UIManager.refreshInventory()
   end)
   ```

3. **애니메이션 표준화**: TweenService 공용 함수
   ```lua
   local function slideIn(frame, from, to, duration)
       frame.Position = from
       frame.Visible = true
       TweenService:Create(frame, TweenInfo.new(duration, Enum.EasingStyle.Back), {
           Position = to
       }):Play()
   end
   ```

### ⚡ 성능 고려

- **오브젝트 풀링**: 슬롯, 알림 등 재사용
- **지연 로딩**: 사용 시점에 UI 생성
- **배치 업데이트**: 여러 슬롯 한번에 갱신
- **가비지 최소화**: 문자열 concat 대신 format
