# Origin-WILD UI/UX 전면 개편 가이드 (v2.0)

본 문서는 플레이어 경험(UX)과 시각적 완성도(UI)를 극대화하기 위한 전면 개편 작업 현황과 향후 로드맵을 정리합니다. 듀랑고(Durango) 스타일의 거친 야생 느낌과 현대적인 반응형 인터페이스의 조화를 목표로 합니다.

---

## 🏗️ 1. 현재 작업 현황 (시스템별 상세)

### 1.1 기술 트리 (K키) - **[완료]**
- **유기적 레이아웃(Organic Tree)**: 고정된 그리드가 아닌 선행 기술 간의 관계(Depth)에 기반한 유동적 트리 구조 구현.
- **카테고리 필터링**: 기초, 도구, 무기, 방어구, 시설 등 카테고리별 기술 분류 기능 추가.
- **연구 상세 정보**: 선택한 기술이 해금하는 제작법(Recipe)과 시설(Facility) 목록을 아이콘과 함께 표시.
- **기술 초기화(Relinquish)**: 전체 기술 연구를 초기화하고 소모한 TP를 전액 환급받는 기능 (서버/클라이언트 동기화 완료).
- **성취감 연출**: 연구 완료 시 화려한 팝업 효과 및 알림 시스템 적용.

### 1.2 장비 및 스탯 (E키) - **[완료]**
- **3D 캐릭터 프리뷰**: 장비창 내 실시간 3D 조명을 적용한 플레이어 캐릭터 모델 배치.
- **회전 조작**: 마우스 드래그 또는 터치 슬라이드를 통해 캐릭터를 360도 회전시켜 장비 착용 샷 확인 가능.
- **실시간 반영**: 인벤토리에서 장비를 장착하거나 교체할 경우 즉시 프리뷰 모델에 시각화.
- **스탯 투자**: 포인트 기반의 스탯 업그레이드 시스템과 각 능력치별 상세 변화치 표시.

### 1.3 소지품 및 인벤토리 (B키) - **[완료]**
- **희귀도 등급 시스템**: 아이템의 Rarity(Common~Legendary)에 따른 슬롯 테두리 색상 차별화.
- **정밀 수량 조절**: 아이템을 버리거나 나눌 때 입력창과 연동되는 **슬라이더(Slider)** UI 도입.
- **반응형 최적화**: PC의 조밀함과 모바일의 조작 편의성을 모두 고려한 가변 슬롯 크기 적용.

### 1.4 메인 HUD - **[진행 중]**
- **액션 클러스터**: 모바일에서 공격, 구르기, 점프, 달리기를 한눈에 조작할 수 있는 헥사곤 스타일 버튼 배치.
- **미니맵 플레이스홀더**: 상단 우측 위치 정보 및 방위 표시용 디자인 적용.
- **경험치 바**: 하단 엣지에 슬림하게 배치된 프로그래스 바와 레벨업 가시성 강화.

## 🚀 2. 향후 작업 로드맵 (Next Steps)

### Phase 1: 빌딩 시스템 고도화
- **건축물 카테고리화**: 기초, 벽, 지붕 등을 휠 메뉴나 탭 메뉴로 선택할 수 있는 전용 UI 구축.
- **건축 미리보기(Ghost)**: 설치 전 가이드라인 표시 로직 연동.

### Phase 2: 상태 효과(Status Effect) 시각화
- **디버프/버프 타이머**: 추위, 배고픔, 무게 초과 등 상태 효과를 아이콘과 남은 시간으로 HUD에 표시.
- **중앙 알림(Toast)**: 중요한 게임 이벤트(야생동물 습격, 점검 등) 알림 연출.

### Phase 3: 미니맵 기능 강화
- **실물 맵 렌더링**: 현재 좌표를 기반으로 주변 지형을 간략하게 렌더링하는 실시간 미니맵 구현.
- **웨이포인트(Waypoint)**: 거점이나 팀원 위치를 미니맵에 표시.

## 🛠️ 3. 기술적 참조 (Rojo & Studio)

| 구분 | 관리 위치 | 주요 모듈 |
| :--- | :--- | :--- |
| **Logic** | `src/Controllers` | `InventoryController`, `TechController` |
| **UI Design** | `src/UI` | `TechUI`, `EquipmentUI`, `HUDUI` |
| **Theme** | `src/UI/UITheme.lua` | 색상 팔레트, 투명도, 폰트 정의 |
| **Utils** | `src/UI/UIUtils.lua` | `mkLine`, `mkBar`, `mkSlot`, `mkSlider` |

---

> **비고**: 모든 UI 요소는 `UIManager.lua`를 통해 중앙 집중식으로 제어되며, `InputManager.lua`를 통해 크로스플랫폼 입력을 지원합니다. 추가 개발이 필요한 요소는 언제든 가이드 수정을 통해 요청해 주세요.
